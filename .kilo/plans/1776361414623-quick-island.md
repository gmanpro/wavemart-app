# Fix Tigrinya Language Issues

## Problem
The user reports that Tigrinya language causes issues when changed - works fine in all other languages.

## Investigation Summary

### Current Architecture
1. **LocaleProvider** (`lib/presentation/providers/app_providers.dart:800-867`):
   - `LocaleNotifier` extends `StateNotifier<LocaleState>`
   - Loads locale from Hive on init
   - `setLocale()` method saves to Hive and updates state
   - Returns `LocaleState.loaded(locale: locale)` after saving

2. **MaterialApp** (`lib/main.dart:67-93`):
   - Watches `localeProvider` and uses `localeState.locale` directly

3. **Settings Screen Language Dialog** (`lib/presentation/screens/settings/settings_screen.dart:326-377`):
   - Opens modal bottom sheet
   - Calls `ref.read(localeProvider.notifier).setLocale()`
   - Then pops the bottom sheet

### ARB Files
- `app_en.arb`: 109 keys
- `app_am.arb`: 109 keys  
- `app_ti.arb`: 109 keys

All 3 languages have complete translations including all UI strings.

## Potential Root Causes

### 1. State Notifier Loading Issue
The `LocaleNotifier` constructor calls `_loadSavedLocale()` which is async:
```dart
LocaleNotifier() : super(const LocaleState.initial()) {
  _loadSavedLocale();  // async, doesn't await
}
```
This could cause timing issues when switching languages quickly.

### 2. State Update Timing
`setLocale()` sets `isLoading: true` then immediately sets to `loaded`:
```dart
Future<void> setLocale(Locale locale) async {
  state = state.copyWith(isLoading: true);  // May trigger rebuild
  final box = await Hive.openBox('app_preferences');
  await box.put('locale', locale.languageCode);
  state = LocaleState.loaded(locale: locale);  // Another rebuild
}
```

### 3. No Error Handling
If Hive fails, state becomes inconsistent - but this would affect all languages.

### 4. Modal Bottom Sheet Rebuild
Using `ref.read()` in the widget, but potential context issues.

## Clarification Received

User confirms the issue is NOT a complete freeze but specific UI elements not responding:
1. Header profile on home screen doesn't respond
2. Settings language doesn't respond  
3. Gray rectangle appears in some areas on the header
4. Whole body of help center doesn't respond

This indicates specific widgets becoming unresponsive after Tigrinya locale is applied, rather than a complete crash. The "gray rectangle" could be an overlay or incomplete render.

## Updated Analysis - New Information

**User confirmed**: "Dialog doesn't open" - when tapping Language in Settings, nothing happens at all.

This is different from the original "unresponsive areas" report. The issue is likely that:
1. The language selection dialog (`showModalBottomSheet`) never triggers
2. OR the settings screen menu itself fails to render properly with Tigrinya

### Root Cause Analysis

Looking at the code flow:
1. Settings screen builds with `AppLocalizations.of(context)` for menu titles
2. When Tigrinya locale is active, if localization fails somehow, the ListView won't build
3. The `_showLanguageSelectionDialog` is a top-level function called from `onTap`

Potential issues:
1. **Async locale loading timing** - `LocaleNotifier` starts with `isLoading: true`, and `_loadSavedLocale()` is async but not awaited
2. **If Hive fails** - state might remain in loading state indefinitely
3. **If `AppLocalizations.of(context)` returns null** - the widget tree might be broken

### Fix Implementation

**Step 1: Fix LocaleNotifier** - Remove intermediate loading state toggle
- File: `lib/presentation/providers/app_providers.dart` lines 830-837

**Step 2: Add null safety in MaterialApp** - Ensure locale is never null
- File: `lib/main.dart` line 80

**Step 3: Add loading state check in Settings** - Show loading indicator while locale loads
- File: `lib/presentation/screens/settings/settings_screen.dart`
